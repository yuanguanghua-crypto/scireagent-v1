## 当前实际路由 vs 建议

### 现状问题

目前的 29 条路由没有分层，知识路径和购物路径混在一起。导航栏上同时出现 Products、Applications、Methods、Protocols、Research Goals 五个平级入口——用户不知道优先走哪个。

### 建议的分层导航结构

顶层导航只有 3 项：

```
[SciReagent Logo]  Products  Knowledge  About  [Sign In] [Register]
                   ↓         ↓
              （下拉菜单）（下拉菜单）
```

**Products 下拉：**
```
Products
├── All Products
├── Nucleotides
├── Click Chemistry
├── Fluorescent Probes
├── Bioconjugation
├── Modifiers
└── Molecular Biology
```

**Knowledge 下拉：**
```
Knowledge
├── Research Goals
├── Applications
├── Methods
├── Protocols
└── References
```

## 逐页跳转细节

### 1. 首页（`/`）

```
出链:
  → 搜索（搜索框提交）
  → 产品详情（点击精选产品卡片）
  → 产品列表（"View all 118 products"）
  → 分类产品列表（点击分类 pill）
  → 知识实体列表（点击知识卡片 → 目标/应用/方法/协议）
  → 登录/注册
```

### 2. 搜索（`/search`）

```
入链: 首页搜索框、全局导航搜索框

页面:
  搜索框（始终在顶部）
  搜索结果分组：
    ┌─ Products (N) ──────────────┐
    │ 结果卡片 → 产品详情          │
    └──────────────────────────────┘
    ┌─ Methods (N) ───────────────┐
    │ 结果卡片 → 方法详情          │
    └──────────────────────────────┘
    ┌─ Protocols (N) ─────────────┐
    │ 结果卡片 → 协议详情          │
    └──────────────────────────────┘

出链:
  → 产品详情
  → 方法详情
  → 协议详情
  →（带 ?q= 参数可分享/收藏搜索结果）
```

### 3. 产品详情（`/products/:id`）— 核心汇合页

```
入链: 搜索、精选产品、产品列表、知识路径（目标→应用→方法→产品）

页面结构（从上到下）:
  ┌─ 面包屑（Research Goal > Application > Method > Product Name）
  ├─ 产品标题 + CAS + 分子式 + 分类标签
  ├─ SKU 表格（价格、规格、库存、加入购物车）
  ├─ 产品描述
  ├─ ⬇ 关联知识（默认折叠）
  │   ├─ 关联方法（方法名 → 链接到方法详情）
  │   ├─ 关联协议（协议名 → 链接到协议详情）
  │   └─ 参考文献
  ├─ ⬇ 兼容产品（默认折叠）
  └─ ⬇ 相关产品

出链:
  → 加入购物车（弹窗提示 → 继续购物 / 去购物车）
  → 购物车
  → 询价单
  → 方法详情（关联方法）
  → 协议详情（关联协议）
  → 其他产品详情（相关产品/兼容产品）
```

**关键改动：关联知识区默认折叠。** 不干扰购物决策，但需要的人可以展开。

### 4. 购物车（`/cart`）→ 结账（`/checkout`）→ 订单

```
购物车:
  入链: 产品详情（加入购物车）、全局导航购物车图标
  出链: → 结账 → 继续购物（← 回到产品页）

结账:
  入链: 购物车
  需要登录（requiresAuth）
  出链: → 订单详情

订单列表/详情:
  入链: 结账成功跳转、全局导航"My Orders"
  需要登录
```

### 5. 知识实体链（ResearchGoal → Application → Method → Protocol）

```
研究目标列表（/research-goals）
  → 研究目标详情（/research-goals/:id）
      标配面包屑
      → 关联应用列表（卡片链接到应用详情）

应用场景列表（/applications）
  → 应用场景详情（/applications/:id）
      → 关联方法列表
      → 关联产品（直接买这个应用领域的产品）

科研方法列表（/methods）
  → 方法详情（/methods/:id）
      → 关联协议列表
      → 关联产品（这个方法需要的试剂）

实验协议列表（/protocols）
  → 协议详情（/protocols/:id）
      → 分步操作指南
      → 关联产品（执行这个协议需要的试剂）
```

### 6. 认证（登录/注册/设置）

```
登录（/login）→ 可带 ?redirect= 参数跳回原页面
注册（/register）→ 多步骤注册（组织选择）
设置（/settings）→ 需要登录
```

## 导航行为规则

| 规则 | 说明 |
|------|------|
| **搜索框全局存在** | 所有页面顶部都有搜索框 |
| **面包屑一致性** | 产品详情页显示完整知识链（目标→应用→方法→产品名） |
| **知识路径不自嗨** | 知识实体的详情页底部必须显示关联产品（可直接购买） |
| **不允许死胡同** | 每个页面至少有一个出链回到产品路径 |
| **管理后台不走前端路由** | `/admin/*` 走 django-unfold，不参与前端导航 |
| **404 兜底** | 保留 NotFound.vue |

## 建议删改的冗余路由

| 当前路由 | 问题 | 建议 |
|----------|------|------|
| `/products/new` 和 `/products/:id/edit` | 与 `/admin/products/new` 和 `/admin/products/:id/edit` 重复 | 只保留 admin 版本，前端编辑走管理后台 |
| `/knowledge-intake` | 管理员批量导入知识的页面，普通用户不需要看到 | 保留但导航栏隐藏入口，仅通过 admin 访问 |
