# SciReagent 实施路线图

## 总原则

- 先让研究员能高效工作、填满数据 → 再优化访客前端体验
- 每一阶段有明确交付物和验收标准
- 不并行做两件事，一次只推进一个阶段

---

## 阶段划分

```
优先度       阶段               交付物              时间
─────────────────────────────────────────────────────────
P0    ┌─── ① Admin 研究员工作台 ── 产品编辑 Inline    1天
      │                         导航精简
      │
P1    ├─── ② 知识导入功能 ──────── Agent JSON 导入器   3-4天
      │                         结构图预览 widget
      │
      ├─── ③ 首页重构 ──────────── 新首页布局        2-3天
      │                         产品详情页优化
      │                         搜索页优化
      │
P2    ├─── ④ 知识链快速创建 ────── 一站式知识录入       2-3天
      │     + CSV 批量导入         CSV 导入器
      │
P3    ├─── ⑤ Hero 背景动画 ──────　精简版知识生长动画   5-7天
      │
P4    ├─── ⑥ 研究员 Dashboard ─── 工作台首页          1-2天
      │
      └─── (持续) 测试 + 修复 ──── 测试覆盖 >50%     持续
```

## 阶段 ① — Admin 研究员工作台（P0，1天）

### 目标

研究员能在**一个页面**上完成产品的增删改查 + 关联知识，不需要跳转。

### 具体改动

| 改动 | 文件 | 改动量 |
|------|------|--------|
| 给 ProductAdmin 加 ProductMethodInline | `commerce/admin.py` | +20 行 |
| 给 ProductAdmin 加 ProductReferenceInline | `commerce/admin.py` | +15 行 |
| 给 ProductAdmin 加 ProductCompatibilityInline | `commerce/admin.py` | +15 行 |
| 给 ProductAdmin 加 ProductProductInline（相关产品） | `commerce/admin.py` | +15 行 |
| SKUInline 调整为带价格的完整行 | `commerce/admin.py` | +5 行 |
| 隐藏 SHUAdmin 独立入口（仅保留 Inline） | `commerce/admin.py` | -2 行 |
| 隐藏 System/Commerce/Accounts 导航组 | `settings/base.py` | -10 行 |
| 添加产品导入入口（CSV 上传，基础版） | `commerce/admin.py` | +40 行 |
| 清理 Relationships 导航分组 | `settings/base.py` | -5 行 |

**总计：~+100 行 / -20 行，纯 Python，不需要改前端。**

### 验收标准

- [ ] 打开产品编辑页，能看到方法、文献、兼容产品、相关产品 4 个 Inline
- [ ] Inline 默认折叠，不干扰产品基础信息的编辑
- [ ] 研究员通过 autocomplete 搜索现有知识实体
- [ ] 保存产品时，关联关系写入桥接表
- [ ] 导航栏只有研究员需要的分组
- [ ] 809 个测试全部通过

## 阶段 ② — 知识导入功能（P1，3-4天）

### 目标

研究员上传 Agent 输出的 `knowledge_graph_v3.json`，系统解析、预览差异、写入数据库。

### 改动

| 改动 | 说明 | 时间 |
|------|------|------|
| 定义 JSON Schema 校验器 | 确保上传的文件格式正确 | 0.5天 |
| 导入顺序编排器 | ResearchGoal → Application → → SKU，依赖解析 | 1天 |
| 差异对比逻辑 | 哪些是新增、哪些是更新、哪些无变化 | 0.5天 |
| Admin 导入界面 | 上传 → 预览 → 确认 → 写入 | 1天 |
| SMILES→结构图预览 widget | 后端 RDKit + 前端 JS 双方案 | 0.5天 |

### 验收标准

- [ ] 上传 v3 JSON，系统正确识别 8 类实体的新增/更新
- [ ] 导入后数据库中的关联关系与 JSON 一致
- [ ] 导入可回滚（出错时不影响已有数据）
- [ ] 结构图在 admin 中实时预览

## 阶段 ③ — 首页重构（P1，2-3天）

### 目标

实现我们之前确定的 5 区域布局 + 英文 + 新设计语言。

### 改动

| 改动 | 说明 | 时间 |
|------|------|------|
| HeroSearch 组件 | 现有的搜索框 + 热门标签 | 0.5天 |
| 分类导航 pills | 从 API 获取分类数据 | 0.5天 |
| 精选产品网格 | 5 列网格，可配置 | 0.5天 |
| 统计栏 | 真实数据（产品数、SKU 数等） | 0.5天 |
| 知识区（折叠） | 默认折叠的知识卡片 | 0.5天 |
| 产品详情页优化 | 购物信息在上，关联知识折叠 | 1天 |
| 搜索页优化 | 统一搜索结果分组展示 | 0.5天 |

### 验收标准

- [ ] 首页展示 5 区域，搜索框占主导
- [ ] 知识区默认折叠，点击展开
- [ ] 英文全站
- [ ] 产品详情页上半屏是购物信息，关联知识可展开
- [ ] 搜索引擎友好的 JSON-LD 仍然正常输出
- [ ] Playwright E2E 测试全部通过
