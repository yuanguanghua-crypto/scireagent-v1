# Executive Summary

| Item | V1.1 | V1.2 RC1 | Reason |
|------|------|----------|--------|

| Search | PostgreSQL FTS mandatory Phase 1 | **Phase 1: icontains** / Phase 1.5: FTS with feature flag | R8: Incremental upgrade |
| Knowledge Graph | Dynamic Cytoscape Phase 1 | **Phase 1: Static SVG** / Phase 3: Dynamic | R9/R16: No Cytoscape.js installed |

| Timeline | Phase 1 = 1 week | **Phase 1 = 3-4 weeks** / Total = 5-7 weeks | R30: Realistic estimate |

# 4. Product Detail Page

## 4.1 Section Order (Fixed)

```
ProductHero
Applications
Compatibility
KnowledgeGraph
Protocols
References
FAQ
RelatedProducts
SKU
RequestQuote
```

## 4.2 Per-Section Specifications

### ProductHero
- Structure Image (SVG from RDKit.js)
- Product Name, CAS, Catalog No, Purity, Storage
- **Sticky Purchase Box** (scroll-fixed): Price, Availability, Add to Basket, Request Quote

### Applications
- Display linked Applications (via ProductMethod bridge)
- Each Application clickable → `/applications/:id`
- Minimum: 1 (hide section if 0)

### Compatibility
- Display related Methods, Protocols, Products
- **Dyes/Enzymes:** Use `category_l1`/`category_l2` as proxy. No new model.
- Data source: `ProductCompatibility` + `ProductProduct` bridge tables

### KnowledgeGraph
- **Phase 1:** Textual relationships (list format)
- **Phase 3:** Interactive Cytoscape.js widget

### Protocols
- Display linked Protocols (via Method → Protocol chain)
- Show: Name, Objective
- **Difficulty:** Not available (Protocol model has no `difficulty` field). Omit or derive from `ProtocolStep.count`.
- **Estimated Time:** Derive from `ProtocolStep.duration_seconds` sum.

### References
- Display linked References (via ProductReference bridge)
- Show: Title, Journal, Year, DOI
- Clickable DOI/PubMed links

### FAQ
- **Runtime-generated** from existing data (no FAQ table)
- **Minimum: 0** — hide section if insufficient data
- **Recommended: 4** questions
- Generation logic:
  ```
  Q1: "What is {name} used for?" → from Application names
  Q2: "Which methods use {name}?" → from Method names
  Q3: "How should {name} be stored?" → from Product.storage
  Q4: "Which protocols use {name}?" → from Protocol names
  ```
- **Fallback:** If data insufficient, hide FAQ section entirely
- **JSON-LD:** Generate FAQPage schema only if FAQ count ≥ 1

### RelatedProducts
- **Priority:** Same Application > Same Method > Same Protocol > Same Category
- **"Same Category"** = same `product_class_id`
- **Algorithm:** Query via ProductMethod bridge, annotate with match priority, order by priority then by name
- **Count:** 4 products (show fewer if insufficient)
- **Data source:** `ProductProduct` bridge table (if populated) or dynamic query

### SKU
- Existing SKU table with enhancements:
  - SKU Code, Pack Size, Concentration, Unit Price, Lead Time, Inventory Status
  - Add to Basket button per row
  - Quantity selector

### RequestQuote
- Collects current basket items
- Submits QuoteRequest form
- See Section 6 for QuoteRequest specification

## 5.2 Phase 1.5: Grouped Search (New Endpoint)

```
GET /api/search/grouped/?q=label+rna
```

Returns grouped format:
```json
{
  "applications": [{"id": 1, "name": "RNA Labeling", "slug": "rna-labeling"}],
  "methods": [{"id": 2, "name": "CuAAC", "slug": "cuaac"}],
  "protocols": [],
  "products": [{"id": 5, "name": "2'-Azido-dATP", "catalog_no": "SC8047"}]

```

**Implementation:** 4 separate Django queries using `SearchVector` + `SearchRank` + `TrigramSimilarity`.

**Feature Flag:**
```python

```

**Dependencies:** `django.contrib.postgres`, `pg_trgm` extension.

## 6.2 New: QuoteRequest (RFQ)

```python
class QuoteRequest(TimeStampedModel):
    class Status(models.TextChoices):
        PENDING = 'pending', 'Pending'
        RESPONDED = 'responded', 'Responded'
        CLOSED = 'closed', 'Closed'

    contact_name = models.CharField(max_length=200)
    contact_email = models.EmailField()
    contact_phone = models.CharField(max_length=30, blank=True)
    organization = models.CharField(max_length=200, blank=True)
    message = models.TextField(blank=True)
    status = models.CharField(max_length=20, choices=Status.choices, default=Status.PENDING)

class QuoteRequestItem(models.Model):
    quote_request = models.ForeignKey(QuoteRequest, on_delete=models.CASCADE, related_name='items')
    product = models.ForeignKey('commerce.Product', on_delete=models.CASCADE)
    sku = models.ForeignKey('commerce.SKU', on_delete=models.SET_NULL, null=True, blank=True)
    quantity = models.IntegerField(default=1)
    notes = models.TextField(blank=True)
```

**API:**
```
POST /api/quote-requests/        → Create quote request
GET  /api/quote-requests/:id/    → View quote request
```

**POST Request Body:**
```json
{
  "contact_name": "Dr. Smith",
  "contact_email": "smith@lab.edu",
  "contact_phone": "+1-617-555-0123",
  "organization": "MIT Chemistry",
  "message": "Need bulk order of 2'-Azido-dATP",
  "items": [
    {"product_id": 5, "sku_id": 12, "quantity": 10},
    {"product_id": 8, "quantity": 5}
  ]
}
```

**No authentication required.** Admin views via Django Admin.

## 7.2 Phase 3: Interactive Graph

**API:**
```
GET /api/graph/?root=product:5&depth=2
```

**Response Schema:**
```json
{
  "nodes": [
    {"id": "app_1", "type": "application", "label": "RNA Labeling", "slug": "rna-labeling"},
    {"id": "method_2", "type": "method", "label": "CuAAC", "slug": "cuaac"}
  ],
  "edges": [
    {"id": "e1", "source": "app_1", "target": "method_2", "relationship": "HAS_METHOD"}
  ],
  "truncated": false
}
```

**Limits:**
```python
MAX_NODES = 50
MAX_EDGES = 100
MAX_DEPTH = 3
TIMEOUT_MS = 500
```

**Relationship Types:**
```
Application → Method:    HAS_METHOD
Method → Protocol:       HAS_PROTOCOL
Product → Method:        USED_IN_METHOD (via ProductMethod bridge)
Product → Protocol:      USED_IN_PROTOCOL (via Method chain)
Product → Product:       RELATED_TO / SUBSTITUTE / COMPLEMENT
```

**Library:** Cytoscape.js (install in Phase 3).
