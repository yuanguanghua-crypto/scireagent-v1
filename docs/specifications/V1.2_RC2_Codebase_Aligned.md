# SciReagent V1.2 RC2 — Codebase-Aligned Specification

**Version:** V1.2 RC2
**Status:** Approved for Implementation
**Supersedes:** V1.2 RC1
**Target Stack:** Django 5.1 + DRF + PostgreSQL + Vue3 + Vite + JavaScript

---

# Executive Summary

V1.2 RC2 aligns the specification with the actual codebase. All API paths, response formats, and data models now match the existing implementation.

## Key Changes from RC1

| Issue | RC1 | RC2 |
|-------|-----|-----|
| API prefix | `/api/` | **`/api/v1/`** |
| Response format | Mixed (bare array / object) | **Envelope `{success, data, meta}` only** |
| Home API | New endpoints | **Extend existing `/api/v1/site/home/`** (additive) |
| Product detail | 7 sub-endpoints | **Single aggregated `/api/v1/products/:id/detail/`** |
| Feature selection | `is_featured` BooleanField | **`display_priority` IntegerField** |
| Featured Solutions refs_count | Included | **Removed** (Protocol.references is TextField) |
| FAQ | Serializer/View | **`faq_service.py` service layer** |
| QuoteRequest | Undefined app | **`apps/quotes`** |
| MethodProtocol | Not mentioned | **Included in entity relationships** |

---

# 1. API Prefix Policy

All API endpoints are mounted under:

```
/api/v1/
```

Examples:
- `/api/v1/site/home/`
- `/api/v1/products/`
- `/api/v1/products/:id/detail/`
- `/api/v1/search/`
- `/api/v1/quote-requests/`

Any examples in this document omitting `/api/v1/` are illustrative only and must not be implemented literally.

---

# 2. Response Envelope Policy

**All APIs** must return the Envelope format:

```json
{
  "success": true,
  "data": {},
  "meta": {}
}
```

No exceptions. No bare arrays. No bare objects.

Existing `EnvelopeMixin` (in `core/mixins.py`) handles this automatically.

## Logic

```python
def generate_faq(product):
    faq = []

    # Q1: What is X used for?
    apps = product.applications_list  # via ProductMethod bridge
    if apps:
        faq.append({
            "question": f"What is {product.name} used for?",
            "answer": ", ".join(app.name for app in apps)
        })

    # Q2: Which methods use X?
    methods = product.methods_list  # via ProductMethod bridge
    if methods:
        faq.append({
            "question": f"Which methods use {product.name}?",
            "answer": ", ".join(m.name for m in methods)
        })

    # Q3: How should X be stored?
    if product.storage:
        faq.append({
            "question": f"How should {product.name} be stored?",
            "answer": product.storage
        })

    # Q4: Which protocols use X?
    protocols = get_protocols_for_product(product)  # via MethodProtocol bridge
    if protocols:
        faq.append({
            "question": f"Which protocols use {product.name}?",
            "answer": ", ".join(p.name for p in protocols)
        })

    return faq
```

**Minimum:** 0 (hide FAQ section if empty)
**Recommended:** 4

# 9. RelatedProducts Algorithm

**Dynamic calculation.** No bridge table needed.

```python
def get_related_products(product, limit=4):
    """Priority: same_application > same_protocol > same_category"""

    # Step 1: Products sharing same Application
    app_product_ids = ProductMethod.objects.filter(
        method__application__in=product.applications_list
    ).values_list('product_id', flat=True).distinct()

    related = Product.objects.filter(
        id__in=app_product_ids
    ).exclude(id=product.id).annotate(
        match_priority=Value(1, output_field=IntegerField())
    )[:limit]

    # Step 2: If不足, add same Protocol
    if related.count() < limit:
        remaining = limit - related.count()
        proto_product_ids = ...  # via MethodProtocol chain
        related |= Product.objects.filter(
            id__in=proto_product_ids
        ).exclude(id=product.id).exclude(
            id__in=related.values_list('id', flat=True)
        ).annotate(
            match_priority=Value(2, output_field=IntegerField())
        )[:remaining]

    # Step 3: If still不足, add same Category
    if related.count() < limit:
        remaining = limit - related.count()
        related |= Product.objects.filter(
            product_class=product.product_class
        ).exclude(id=product.id).exclude(
            id__in=related.values_list('id', flat=True)
        ).annotate(
            match_priority=Value(3, output_field=IntegerField())
        )[:remaining]

    return related.order_by('match_priority', 'name')
```
